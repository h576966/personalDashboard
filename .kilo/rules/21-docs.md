# Documentation

Apply these rules when working with documentation files.

## Do Not Create Without Being Asked

- Do not create README files, markdown docs, or any documentation unless explicitly requested.
- Do not proactively generate API docs, guides, or changelogs.
- When asked to create docs, keep them concise and structured.

## Keep Documentation in Sync

- When you change code that is documented, update the documentation in the same change.
- Stale docs are worse than no docs. If you notice outdated documentation, flag it.

## Formatting

- Use GitHub-flavored markdown.
- Use fenced code blocks with language tags for all code snippets.
- Keep lines at a reasonable length for reviewability in diffs (under 120 characters).
